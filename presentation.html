Digital Twin Factory System

Prepared By: Ritik Verma
Date: 2024-06-09

Introduction & Motivation

  • Modern factories require real-time visibility and control for efficiency and quality.
  • Traditional monitoring is fragmented and lacks a unified, interactive view.
  • Digital Twin: A virtual replica of the factory, synchronized with live data, enabling monitoring, simulation, and optimization.

Project Objective

  • Develop a Digital Twin for a factory using a web-based 3D environment.
  • Integrate real-time PLC and order data to drive the virtual scene and dashboards.
  • Enable users to monitor, interact with, and analyze factory operations virtually.

System Overview

  • Frontend: React + @react-three/fiber for 3D visualization and dashboards.
  • Backend: FastAPI server with MongoDB, providing real-time data via WebSocket.
  • Data Sources: PLCs, production orders, inventory, and process steps.
  • Synchronization: Every real-world event is mirrored in the digital twin UI.

Key Features

  • 3D interactive factory scene rendered with React and @react-three/fiber.
  • Realistic models: Cobot (robot arm), AGV (automated guided vehicle), racks, conveyor, stations, human avatars, WiFi animation, and more.
  • Real-time PLC and order data drive all animations and UI updates.
  • Configurable process steps loaded from CSV, with step-by-step navigation and visual feedback.
  • Color-coded racks and inventory, with dynamic updates and low-quantity alerts.
  • Live dashboards for orders, assembly, PLC status, and inventory, all updated via WebSocket.
  • User can control and monitor process steps, trigger animations, and view order flows.
  • Panel for minimizing/maximizing controls and persistent state (localStorage) for rack quantities and selections.

Digital Twin Factory Scene: 3D Setup

  • The scene includes a full factory floor with lighting, camera controls (zoom, pan, orbit), and physics for realism.
  • Cobot model supports multiple animation sequences (pick-up, scan, etc.), triggered by PLC data or user actions.
  • AGV model moves along a defined path, with animation speed and state controlled by backend data.
  • Racks are grouped by station, each rack is color-coded and can be highlighted or updated based on process steps or inventory.
  • Conveyor is animated to show part movement, and can be started/stopped by PLC signals.
  • Multiple stations (Station 1, 2, 3) each have their own models, animations, and process logic.
  • Human avatars (Character1, Character2) add realism and context to the digital twin.
  • Additional assets: WiFi animation for connectivity status, rotating cylinders, platforms, and more.

Real-Time Data Flow & Synchronization

  • WebSocket connection to FastAPI backend streams PLC, order, and assembly data in real time.
  • Every PLC event (e.g., conveyor start/stop, cobot pick-up/scan) triggers corresponding 3D animation and UI update.
  • Order and assembly dashboards update instantly as new data arrives.
  • Configurator panel allows users to step through the process, with each step updating the 3D scene and inventory.
  • Low inventory alerts are shown immediately when rack quantities fall below threshold.
  • AllStations and other models subscribe to live data and update their state/appearance accordingly.
  • Animations are paused/resumed based on process state, and can be controlled by both backend data and user input.
  • Persistent state (e.g., rack quantities, manual selections) is saved in localStorage for continuity across sessions.

User Interaction & Visualization

  • Users can interact with the 3D scene: zoom, pan, orbit, and select elements.
  • Configurator panel provides step-by-step process navigation, rack color selection, and inventory management.
  • Live dashboards (OrderStats, AssemblyStats, PLCUpdates, RecentOrders) visualize all key factory metrics.
  • Clicking on certain 3D elements (e.g., KizunaLink) opens external dashboards for deeper analysis.
  • Alerts and feedback are shown for low inventory, process events, and connection status.
  • Users can minimize/maximize the control panel for a focused 3D view.
  • Animations and process steps can be triggered manually or by live PLC data.
  • The UI is responsive and adapts to different screen sizes for accessibility.

Extensibility & Future Scope

  • Modular architecture allows easy addition of new 3D models, sensors, or process logic.
  • Supports integration with additional data sources (IoT, ERP, MES) for a more comprehensive digital twin.
  • Can incorporate AI/ML modules for predictive maintenance, anomaly detection, or process optimization.
  • Simulation and what-if analysis can be added for process improvement and training.
  • UI and dashboards can be extended with more KPIs, historical trends, and advanced analytics.
  • The system can be deployed on-premises or in the cloud for scalability.
  • Designed for real-world factory environments, but adaptable to other domains (e.g., logistics, warehousing).

Results & Impact

  • Unified, interactive view of the factory floor and operations.
  • Instant visibility into process status, inventory, and orders.
  • Reduced manual monitoring and improved responsiveness.
  • Foundation for advanced analytics and smart factory initiatives.
