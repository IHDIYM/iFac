Introduction & Motivation

  • Modern factories require real-time visibility and control for efficiency and quality.
  • Traditional monitoring is fragmented and lacks a unified, interactive view.
  • Digital Twin: A virtual replica of the factory, synchronized with live data, enabling monitoring, simulation, and optimization.
C4i4
7/22/2025 | C4i4

Project Objective

  • Develop a Digital Twin for a factory using a web-based 3D environment.
  • Integrate real-time PLC and order data to drive the virtual scene and dashboards.
  • Enable users to monitor, interact with, and analyze factory operations virtually.
C4i4
7/22/2025 | C4i4

System Overview

  • Frontend: React + @react-three/fiber for 3D visualization and dashboards.
  • Backend: FastAPI server with MongoDB, providing real-time data via WebSocket.
  • Data Sources: PLCs, production orders, inventory, and process steps.
  • Synchronization: Every real-world event is mirrored in the digital twin UI.

3D Digital Twin Setup

  • Full factory floor with lighting, camera controls, and physics for realism.
  • Cobot and AGV models with live, data-driven animations.
  • Color-coded racks and stations update with process steps and inventory.
  • Conveyor and other assets animate based on PLC signals.
  • Human avatars and WiFi animation add context and realism.
C4i4
7/22/2025 | C4i4

Real-Time Data Flow & Synchronization

  • WebSocket streams real-time PLC and order data to the digital twin
  • Every process event triggers instant 3D animation and dashboard updates
  • Configurator panel steps update the scene and inventory live
  • Low inventory and process alerts appear immediately
  • All models and dashboards stay in sync with factory data
